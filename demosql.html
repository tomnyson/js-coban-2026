Lý Thuyết MySQL

MySQL là một trong những hệ quản trị cơ sở dữ liệu quan hệ (RDBMS) phổ biến nhất thế giới.

1. MySQL là gì?

MySQL là hệ quản trị cơ sở dữ liệu quan hệ mã nguồn mở (open-source RDBMS), được phát triển bởi Oracle. Nó sử dụng ngôn ngữ SQL để quản lý dữ liệu. MySQL hỗ trợ nhiều hệ điều hành và được sử dụng rộng rãi trong các ứng dụng web như WordPress, Facebook.

2. RDBMS & Các khái niệm chính

  • Table (Bảng): Lưu trữ dữ liệu theo hàng (rows) và cột (columns), giống như bảng Excel.
  • Primary Key: Khóa chính, giá trị duy nhất để xác định mỗi bản ghi, không thể null.
  • Foreign Key: Khóa ngoại, tạo mối quan hệ giữa các bảng để tránh dữ liệu trùng lặp và đảm bảo tính nhất quán.
  • SQL: Structured Query Language – Ngôn ngữ chuẩn để truy vấn, cập nhật và quản lý dữ liệu.
  • Normalization: Quá trình tổ chức dữ liệu để giảm dư thừa, thường áp dụng các dạng chuẩn 1NF, 2NF, 3NF.

3. Các lệnh MySQL cơ bản

CREATE DATABASE QL_SINHVIEN;

CREATE TABLE SINH_VIEN (MaSinhVien CHAR(7) PRIMARY KEY, HoTen NVARCHAR(30));

SELECT * FROM SINH_VIEN WHERE Lop = 'CNTT01';

INSERT INTO SINH_VIEN VALUES ('SV001', 'Nguyen Van A', 'CNTT01');

UPDATE SINH_VIEN SET HoTen = 'Nguyen Van B' WHERE MaSinhVien = 'SV001';

DELETE FROM SINH_VIEN WHERE MaSinhVien = 'SV001';

4. Joins, Indexes và Normalization

MySQL hỗ trợ các truy vấn phức tạp để xử lý dữ liệu từ nhiều bảng:

  • INNER JOIN: Kết hợp bản ghi khớp từ hai bảng dựa trên điều kiện.
  • LEFT JOIN: Bao gồm tất cả bản ghi từ bảng trái, ngay cả khi không khớp.
  • Indexes: Tạo chỉ mục trên cột để tăng tốc độ tìm kiếm và JOIN, nhưng có thể làm chậm INSERT/UPDATE.
  • Normalization vs Denormalization: Normalization giảm dư thừa; denormalization tăng hiệu suất cho read-heavy apps.

SELECT s.HoTen, bd.Diem FROM SINH_VIEN s INNER JOIN BANG_DIEM bd ON s.MaSinhVien = bd.MaSinhVien;

CREATE INDEX idx_lop ON SINH_VIEN(Lop);
